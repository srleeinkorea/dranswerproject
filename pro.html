보호자 입력 데이터를 주간 단위로 설계한 임상적 · 운영적 근거

왜 일일이 아닌 주간인가?

PRO는 단순 편의성 설계가 아닙니다. 임상 효과성, 보호자 순응도, 데이터 품질, 시스템 효율성을 모두 고려한 전략적 결정입니다.

데이터 소스 수집 주기 특성 역할
Ventilation Signal 1시간 객관적 · 자동화 급성 변화 감지
PRO Data 1주일 주관적 · 구조화 입력 만성 추세 파악

주간 설정의 핵심 이유

보호자 부담 최소화

일일 입력은 순응도 저하를 초래. 주간 1회는 지속 가능한 입력 구조.

데이터 품질 향상

의도적이고 체계적인 평가가 산발적 입력보다 임상적 가치가 높음.

벤틸 신호와의 보완성

고주파 생리 데이터 + 저주파 주관적 데이터의 균형.

시스템 효율성

중복 데이터 축적 방지 및 ML 학습 안정성 확보.

임상 워크플로우 통합

  • 매시간 벤틸 신호 자동 수집
  • 주 1회 보호자 구조화 평가
  • 긴급 이벤트는 즉시 보고 채널로 분리
  • 7일 누적 데이터 통합 분석
결론: 자동화된 고빈도 생리 데이터와 의도적 저빈도 임상 평가의 균형이 가장 안정적인 모니터링 구조입니다.

샘플 앱: 주간 PRO 입력 화면

아래는 “지난주와 동일(기본값) + 변화만 탭(칩) + 체온/SpO₂ 별도 입력 + 메모 최소화” 원칙을 반영한 샘플 화면입니다. 실제 서비스에서는 저장/리마인드/이상 소견 대응 UX가 정책에 따라 연결됩니다.

설계 포인트

  • 기본값을 “지난주와 비슷”으로 설정하여 입력량 최소화
  • 주 1회에 적합한 “변화형” 응답(증가/감소/비슷) 중심
  • 체온, SpO₂는 별도 입력 카드로 분리(가시성/일관성 강화)
  • 알람/이벤트는 “있었음” 선택 시에만 상세 항목 노출(조건부 UX)
목표 입력 시간 30–60초
핵심 문항 수 6문항 + (조건부 상세)
입력 방식 원탭 칩 + 최소 숫자
피로도 전략 기본값/조건부/옵션 메모
운영 측면에서는 “미완료 자동저장”, “1회 리마인드”, “이번 주 생략(이유 선택)”을 함께 두면 이탈을 크게 줄일 수 있습니다.

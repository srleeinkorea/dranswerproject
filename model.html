Patient Monitoring Analytics Architecture

본 시스템은 환자의 미래 악화를 예측하는 모델이 아니라, 최근 상태 데이터를 정량화하고 변화 패턴을 분석하는 Monitoring Analytics 기반 구조입니다.

System Architecture

1️⃣ Data Collection

• Ventilation Signal (1시간 단위)
• PRO Weekly Input
• Event Trigger Reporting

Structured Clinical Data

2️⃣ Monitoring Analytics Engine

• 개인 기준선 계산 (Baseline Modeling)
• Z-score / 변동성 분석
• Trend Detection (7일 이동 기울기)
• Composite Monitoring Index 생성

No Future Prediction

3️⃣ Rule & Triage Layer

• Green / Yellow / Red
• 운영 레벨 분류
• 의료진 확인 권고 로직

Operational Decision

4️⃣ LLM Interface

• 상태 설명 자연어 변환
• 진단/처방 금지 가드레일
• PRO 입력 구조화
• 의료진 연결 안내

Communication Layer

Triage Definition (Monitoring Level)

Level 의미 특성 운영 조치
🟢 Green 기준선 범위 내 안정 편차 낮음 · 변동성 낮음 일상 모니터링 유지
🟠 Yellow 상태 변화 감지 추세 악화 · 변동성 증가 관찰 강화 · 의료진 공유
🔴 Red 즉시 확인 필요 급격한 편차 · 이벤트 동반 의료진 확인 권고

Monitoring Output Example

stability_index: 0.68 deviation_score: +1.8σ trend_flag: "7일간 점진적 상승" attention_factors: - RR_variability 증가 - MinVent 감소 - Leak spike 2회 발생 triage_level: YELLOW

LLM Role (Non-Clinical Decision)

LLM은 의료 판단을 수행하지 않습니다. Monitoring Analytics Engine에서 계산된 결과를 환자 친화적 언어로 설명하고, 진단·치료 관련 표현을 자동 차단하며, 필요 시 의료진 확인을 안내합니다.

입력: "요즘 숨이 조금 가빠 보여요" 응답: "최근 48시간 동안 호흡 패턴의 변동성이 증가했습니다. 현재는 관찰 강화 단계로 분류되며, 의료진과 상황을 공유하는 것이 권장됩니다. 증상이 악화되면 즉시 알려주세요."

Monitoring, Not Prediction.

이 시스템은 환자의 미래를 예측하지 않습니다. 현재와 최근 데이터를 기반으로 상태를 분석하고, 운영적 의사결정을 지원합니다.

보호자 입력 데이터를 주간 단위로 설계한 임상적 · 운영적 근거

왜 일일이 아닌 주간인가?

PRO는 단순 편의성 설계가 아닙니다. 임상 효과성, 보호자 순응도, 데이터 품질, 시스템 효율성을 모두 고려한 전략적 결정입니다.

데이터 소스 수집 주기 특성 역할
Ventilation Signal 1시간 객관적 · 자동화 급성 변화 감지
PRO Data 1주일 주관적 · 구조화 입력 만성 추세 파악

주간 설정의 핵심 이유

보호자 부담 최소화

일일 입력은 순응도 저하를 초래. 주간 1회는 지속 가능한 입력 구조.

데이터 품질 향상

의도적이고 체계적인 평가가 산발적 입력보다 임상적 가치가 높음.

벤틸 신호와의 보완성

고주파 생리 데이터 + 저주파 주관적 데이터의 균형.

시스템 효율성

중복 데이터 축적 방지 및 ML 학습 안정성 확보.

임상 워크플로우 통합

  • 매시간 벤틸 신호 자동 수집
  • 주 1회 보호자 구조화 평가
  • 긴급 이벤트는 즉시 보고 채널로 분리
  • 7일 누적 데이터 통합 분석
결론: 자동화된 고빈도 생리 데이터와 의도적 저빈도 임상 평가의 균형이 가장 안정적인 모니터링 구조입니다.

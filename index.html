시스템 개요

본 시스템은 장기 기계호흡 환자의 안정적 관리를 위해 설계되었습니다. 홈 벤틸레이터로부터 수집한 호흡 신호와 보호자가 제공하는 임상 정보를 통합 분석하여 환자의 상태 변화를 감지하고 적절한 임상 개입을 지원합니다.

Ventilation Signal – 1시간 주기 Hourly
PRO Data – 보호자 주간 입력 Weekly
Event Trigger – 긴급 상황 판단용 Realtime

AI 모델 아키텍처 – 이중 구조

위험도 산출과 설명 인터페이스는 서로 다른 인공지능 모델이 담당합니다. 임상적 상태 평가는 수치 기반 분석 모델이 수행하며, LLM은 해당 결과를 설명하는 보조 역할을 수행합니다.

① 모니터링 & 상태 분석 모델
Deep/Machine Learning

벤틸레이션 신호(PIP, RR, VTe, MinVent, Leak)와 PRO 데이터를 통합하여 환자의 현재 위험도를 산출합니다.

  • Gradient Boosting (LightGBM, XGBoost)
  • 시계열 보조 모델(RNN/LSTM 가능)
  • 7일 추세 기반 안정성 분석
  • Green / Amber / Red 분류

→ 상태 평가 & 위험도 분류 중심 구조

② LLM 인터페이스 엔진
Large Language Model

위험도 분석 결과를 자연어로 설명하고, 보호자 및 의료진이 이해하기 쉬운 형태로 전달합니다.

  • 위험도 결과 요약
  • 상태 변화 설명
  • 응급 상황 안내 문구 생성
  • 사용자 질문 응답

→ 진단 또는 치료 판단을 수행하지 않는 설명 보조 인터페이스

벤틸레이션 신호 분석

데이터 수집 프로토콜

호흡 신호는 1시간 단위로 정기적으로 수집되어 안정화된 상태의 환자 데이터를 확보합니다. 이러한 간격은 일시적 변동을 최소화하고 임상적으로 의미 있는 패턴 변화를 감지하는 데 최적화하도록 설계합니다.

데이터 처리 방법론

수집된 신호 데이터는 다단계 처리 프로세스를 거칩니다. 먼저 기기 노이즈나 순간적인 이상치를 제거하고, 환자의 기준값 대비 상대적 변화도를 계산합니다. 이후 지난 7일 데이터를 기반으로 추세 분석을 수행하여 점진적인 악화 신호를 감지합니다.

Traffic Light Triage System

머신러닝 기반 위험 점수는 임상적으로 해석 가능한 3단계 분류 체계로 변환됩니다.

  • 🟢 Green
    : 신호 안정적, 일상 모니터링 유지
  • 🟠 Amber
    : 변화 감지, 강화된 관찰 필요
  • 🔴 Red
    : 임상적 위험 신호, 의료진/응급실 권고

LLM 응답 예시

LLM은 위험도 산출 결과를 기반으로 자연어 설명을 생성합니다. 의료 판단을 수행하지 않으며, 분석 결과 전달에 한정됩니다.

입력: "요즘 호흡 상태가 어떤가요?"

분석 결과: Amber

응답:
"최근 호흡 패턴에서 변화가 감지되었습니다. 의료진과 상태를 공유하고 추가 관찰을 권장합니다. 급성 증상이 발생하면 즉시 연락하십시오."
